pnpm workspace; all packages ship tests. Use pnpm --filter @redesigner/<pkg> ... not cd packages/<pkg> && npm .... Root package.json intentionally has no test script; use pnpm -r run test or pnpm --filter.
Five workspace packages: @redesigner/core, @redesigner/vite, @redesigner/mcp, @redesigner/daemon, @redesigner/ext. Downstream packages (vite, mcp, daemon, ext) depend on core via workspace:* and resolve it through packages/core/dist/ (the exports field points at compiled JS, not source). Rebuild with pnpm --filter @redesigner/<pkg> build before typechecking or testing consumers after source changes. Vite spawns daemon via child_process.fork at dev-server start; mcp discovers the running daemon via handoff file under the OS runtime dir; ext is an MV3 Chrome extension that talks to the daemon over WS+REST.
ESM-only repo (type: module, Node ≥ 22 — constrained by ts-json-schema-generator@2.9 and @vitejs/plugin-react@5). Use import.meta.dirname, not __dirname.
@babel/traverse and @babel/generator publish CJS; under ESM interop they expose their real default as .default.default. Pattern: const traverse = (traverseMod as any).default ?? traverseMod. Missing this silently no-ops on the first call.
Vitest cannot vi.spyOn node:fs or node:fs/promises — the ESM namespace is frozen. Use vi.mock('node:fs', async (importOriginal) => ({ ...await importOriginal() })) to unfreeze, or vi.hoisted to share a recorder across mock factory and assertions.
Zod is pinned to ^3.25.0 || ^4.0.0 but @modelcontextprotocol/sdk resolves v4. Internal schema._def.shape and _def.typeName exist on v3 only; hand-rolled converters silently emit empty JSON Schemas on v4. Use z.toJSONSchema() and strip the $schema key.
tsup configs with banner: { js: '#!/usr/bin/env node' } already inject the shebang. Do not also write #!/usr/bin/env node at the top of src/cli.ts or the build emits two shebangs and Node parses the second as ESM syntax error.
Do not hand-edit or hand-format fixture outputs at packages/vite/test/fixtures/**/output.tsx and expected-manifest.json. They are Biome-ignored and tsc-excluded. Regenerate via REDESIGNER_FIXTURE_UPDATE=1 pnpm --filter @redesigner/vite run test:fixtures and add a line to packages/vite/test/fixtures/FIXTURE_CHANGELOG.md — the pre-commit hook blocks fixture edits without one.

AbortSignal.timeout(ms) in daemon + mcp + ext SW fetch callers — never new AbortController + setTimeout (undici#2198 EventTarget leak on long-lived callers). Also never AbortSignal.any([timeout, userSignal]) — composes the same leak class (node#57736).
process.channel?.unref() after registering process.on('disconnect', ...) in packages/daemon/src/child.ts — otherwise IPC channel pins event loop independent of HTTP+WS servers. Also: no other process.on('message', ...) listener anywhere in daemon code; adding one re-refs the channel and silently reverts this.
crypto.timingSafeEqual throws RangeError on length mismatch. Daemon's compareToken helper normalizes inputs to equal length before compare. If you add a new auth path, route it through that helper.
fs.watch on macOS silently degrades under Spotlight/Time Machine load; daemon's ManifestWatcher compensates via 3s stat-poll fallback. If anyone rips that out thinking it's redundant, expect flakiness reports from macOS users.
Zod schemas in daemon + ext code must be module top-level; in-handler z.object(...) is a known 100x Zod v4 regression cliff.
CRXJS is pinned to ~2.4.0 (tilde, not caret) in packages/ext/package.json. 2.5+ changes manifest transform semantics and breaks the content_scripts ordering plus the `key` field passthrough. If you bump it, also walk through packages/ext/docs/ext-build-migration.md.
packages/ext/manifest.json has a `key` field (RSA SPKI public key). It locks the unpacked extension ID so TOFU pinning in the daemon + storage.session key derivation survive reloads. Losing or rotating it changes the ext ID and invalidates the ID allowlist on the daemon side.
.zod-version at repo root is the major-pin sentinel. The CI guard in .github/workflows/ci.yml parses pnpm-lock.yaml for the resolved zod major and diffs against .zod-version; bumping Zod major requires updating this file and reviewing packages/core/test/schemas-parity.test.ts + packages/ext/test/contract/goldens.test.ts which encode v3↔v4 detection via typeof z.toJSONSchema.
Playwright E2E: smoke spec runs in CI as a launch-only harness sanity check; nightly specs under packages/ext/test/e2e/nightly/ early-return unless PW_FULL_HARNESS=1 is set. This is intentional — full harness (daemon + vite + ext dist) is post-v0 work. Do not "fix" the early-returns by turning them into test.skip (the implementer verified that conflicts with biome noSkippedTests semantics).

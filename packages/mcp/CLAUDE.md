MCP-local gotchas for contributors.
Integration tests in test/integration/ require dist/cli.js to exist before running. CI pipeline MUST run pnpm -r build before pnpm -r test. Opt-in REDESIGNER_MCP_BUILD_IN_TEST=1 triggers in-test rebuild via spawnSync (never shell exec).
test/integration/selection-roundtrip.test.ts spawns the CLI via StdioClientTransport which uses cross-spawn without detached:true — child inherits our process group. Kill with positive-PID SIGKILL (never `-pid` group-kill — that would hit vitest itself). Dual beforeExit+exit guards handle Vitest tinypool force-terminate (vitest#3077). StdioClientTransport.close() only sends SIGTERM (SDK#579); reap via once(childProc,'exit') with 1s ceiling before SIGKILL fallback.
SDK's getDefaultEnvironment() strips TMPDIR (and most env vars) — pass `env: { ...process.env }` when constructing StdioClientTransport, otherwise macOS child processes lose the TMPDIR override and os.tmpdir() falls back to /tmp, which breaks handoff discovery (handoff path is keyed on os.tmpdir() + com.redesigner.<uid>/<projectHash>).
Handoff path derivation in daemonBackend.ts resolveHandoffPath is the source of truth: linux=$XDG_RUNTIME_DIR/redesigner/<hash>, darwin=os.tmpdir()/com.redesigner.<uid>/<hash>, win32=$LOCALAPPDATA/redesigner/<uid>/<hash>. If replicating in a test, grep the real function first — drift will break silently (MCP just shows daemon as unreachable).

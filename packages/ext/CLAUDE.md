Extension-local gotchas for contributors.
chrome.storage.session requires setAccessLevel({accessLevel: 'TRUSTED_CONTEXTS'}) called at SW boot. Wrap in try/catch — older Chrome throws and we degrade to re-exchange on wake rather than persist fast-path keys.
Session key name is opaque per-boot: s_<crypto.randomUUID()>. Cache it on globalThis.__sessionKey so wake-ups reuse the same key for the life of the installed extension process tree. Do not persist the key name to local storage.
All chrome.runtime + chrome.storage + chrome.action + chrome.alarms addListener calls in the SW must register synchronously before any await. Chrome drops events delivered to a suspended SW that hasn't registered the listener yet, producing silent first-event loss after a wake.
globalThis.__bootEpoch is incremented synchronously on every SW module load. Tasks that survive wake (pending RPCs, in-flight exchanges) must capture the epoch at insertion time and bail on resolve if the epoch moved, to avoid resolving stale callbacks from a previous boot.
useSyncExternalStore in the panel requires useCallback-stable subscribe + getSnapshot references, and must pass getServerSnapshot === getSnapshot. Without stable refs React 19 enters a strict-identity churn loop that surfaces as "Maximum update depth exceeded".
Picker overlay: shadow host uses popover="manual" + showPopover() for top-layer promotion. Capture-phase listeners must use stopImmediatePropagation (not stopPropagation) to beat monaco/codemirror/tldraw handlers. Open shadow roots are traversed recursively by hitTest; closed shadow roots remain opaque and we return the host element.
WS subprotocol bearer is `base64url.bearer.authorization.redesigner.dev.<token>` paired with `redesigner-v1` for version negotiation. The prefix format is k8s-style and is not arbitrary — packages/daemon/src/auth.ts `SUBPROTO_BEARER_PREFIX` is the source of truth.
Grace window on wake: min(15000, max(3000, now - sw.wakeAt)). Pending RPCs sweep past-deadline only after this grace elapses. Do not shortcut it to a fixed constant — the bounds protect against wake-time clock skew as well as long-running RPCs.
